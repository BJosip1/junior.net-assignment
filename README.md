Aplikacija se pokreće pomoću Ctrl+F5 tipki odnosno Start without Debugging. Launch profile selector treba biti na https.

&#x20;Aplikacija ima 1 kontroler koji se zove RestaurantController. Pomoću njega korisnik može preko Get metode dobiti listu svih narudžbi ako je sortByTotal postavljen na false, listu svih narudžbi poredane po padajućem nizu ako je sortByTotal postavljen na true. Tada se gleda ukupan iznos računa. Post metoda služi kako bi korisnik unio podatke za narudžbu. To su ime, adresa, broj, napomena, valuta kojom plaća, metoda plaćanja te ime jela, količina i cijena. Postoji Get metoda koja traži samo po id određenu narudžbu. Put metoda služi kako bi se preko id narudžbe mijenjao status narudžbe. Postoji 3 statusa a to su: Na čekanju, Priprema te Završeno. Postoje 2 načina plaćanja Gotovina i kartica. Za načine plaćanja upisuje 1 što znači gotovina ili 2 što znači kartica. Promjena statusa narudžbe također ide preko id gdje je id=1 znači "Na čekanju", kad je id=2 to znači "Priprema", id=3 znači "Završeno". U Domain se nalaze 4 modela, u Application se nalazi interfaces od IOrderService i IApplicationDbCContext. U Application se također nalaze DTO i OrderService kao i Common koji sadrži Result i ValidateResult  koji prikupljaju greške i šalju ih od servisa do kontrolera. Abysalto.Junior sadrži Program.cs, konfiguracijju kako modeli u bazi izgledaju odnosno koja su im ograničenja te ApplicationDbContext, kontroler i StatusHandler koji pretvara Result<T> u http odgovor i govori je li status npr. 200 ili 400.
